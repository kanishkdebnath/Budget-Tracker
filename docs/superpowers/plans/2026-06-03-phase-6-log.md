# Phase 6 — Log (F1) — Implementation Plan

> REQUIRED SUB-SKILL: superpowers:executing-plans. Calculator (F8) is **Phase 6b**.

**Goal:** Replace the Log placeholder with the real transaction log for the shared month — Net band, per-date transaction cards, add/edit/delete, `Money.parseToMinor` amount entry, and the F1.5 cross-month jump.

**Architecture:** `LogViewModel` holds the viewed month (`setMonth` via `flatMapLatest`) and combines `TransactionRepository.observeMonth` + categories + groups + a filter into a pure, unit-tested `buildLogState` (month Net-band totals always; the filter only narrows per-date sections). Amounts are `Long` minor units formatted via `Money.format`; the amount field parses with `Money.parseToMinor` (§F1.6). `NetBand` is a shared component (Plan/Report reuse it). The month is owned by `BudgetApp`; `LogScreen` calls `onMonthChange` for §F1.5.

## Files
- `domain/money/Money.kt` — `parseToMinor` (§F1.6) + `symbolOf`.
- `ui/screens/log/LogViewModel.kt` — `TxnFilter`, `TxnRow`, `DateSection`, `LogUiState`, pure `buildLogState`, VM (setMonth/filter, add/edit/delete).
- `data/repository/TransactionRepository.kt` — `edit(id, …)` preserving `createdAt`.
- `ui/AppViewModelProvider.kt` — `LogViewModel` initializer.
- `ui/components/NetBand.kt` — shared Income·Expense·Net band.
- `ui/screens/log/LogComponents.kt` — `LogFilterChips`, `DateCard`, transaction row.
- `ui/screens/log/TransactionSheet.kt` — add/edit form (amount parse, category dropdown, M3 date picker, note, delete).
- `ui/screens/log/LogScreen.kt` — Net band + filter chips + per-date cards + FAB + sheet + §F1.5.
- `ui/BudgetApp.kt` — pass `month` + `onMonthChange` to `LogScreen`.
- tests: `MoneyTest` parse cases, `LogStateTest` (totals/sections/filter).
