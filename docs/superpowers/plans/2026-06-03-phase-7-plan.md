# Phase 7 — Plan (F3) — Implementation Plan

> REQUIRED SUB-SKILL: superpowers:executing-plans.

**Goal:** Replace the Plan placeholder with per-category monthly targets — live target Net band, carry-forward pre-fill, atomic bulk save.

**Architecture:** `PlanViewModel` holds the viewed month + a `categoryId → text` input map (the edit buffer). On `setMonth` it loads the month's targets; if none, it pre-fills from the previous month (§F3.4) and sets a banner. Target Net-band totals come from a pure `computeTargetTotals(inputs, categories)`. Save writes every live category's parsed input via the atomic `TargetRepository.bulkSave` (blank/invalid clears, §F3.3/§3.5). `NetBand` (Phase 6) is reused.

## Files
- `domain/money/Money.kt` — `parseTargetToMinor` (allows 0) + `toMajorInput`.
- `ui/screens/plan/PlanViewModel.kt` — `PrefillBanner`, `PlanGroup`, `PlanTotals`, pure `buildPlanGroups`/`computeTargetTotals`, VM (setMonth/load+carry-forward, onInputChange, save).
- `ui/AppViewModelProvider.kt` — `PlanViewModel` initializer.
- `ui/screens/plan/PlanComponents.kt` — `PlanBanner`, `PlanGroupCard` (per-category target input), `PlanSaveBar`.
- `ui/screens/plan/PlanScreen.kt` — banner + target `NetBand` + grouped inputs + sticky Save bar + snackbar.
- test: `PlanLogicTest` (totals by kind, blank/invalid = 0, `parseTargetToMinor` 0/blank/negative).
